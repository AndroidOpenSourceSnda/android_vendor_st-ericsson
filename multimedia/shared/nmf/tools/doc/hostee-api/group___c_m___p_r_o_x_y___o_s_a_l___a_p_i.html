Detailed Description

OSAL CM Proxy OSAL (Operating System Abstraction Layer) API.

This file contains the OSAL API for CM proxy that must be implemented by integrator for each user or kernel application that access CM API. This CM API was provided through a CM user or kernel proxy.

PUBLIC t_nmf_osal_fs_file_handle OSAL_ReadFile ( const char *  fileName,
void **  buffer,
t_cm_size *  pSize,
t_bool  userspace 
)

Description of the File System part of the OS Adaptation Layer.

Goal: Load dynamically the component file from the component repository. Here, it loads the file and read the data into a buffer

Called (indirectly) by:

  • CM_RepositoryInit() call once at initialization of the client
  • CM_InstantiateComponent() call (only for the first user component instantiation and when loading the EE on a given Media Processor)
  • CM_BindComponentxxx() call (only for the first binding component instantiation)
Parameters:
[in] fileName,: (const char *) path of the file to be checked
[out] buffer,: (void **) pointer to the data read (content of the file)
[out] pSize,: (t_cm_size *) pointer to the size of the data read
[in] userspace,: (t_bool) boolean to specify whether the handler/pointer returned will be used in userspace or kernelspace
Returns:
file handle on the file opened or Null in case of error
