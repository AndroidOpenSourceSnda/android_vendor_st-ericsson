Trace Viewer Workbench

Documentation > References > Console View


Description

The console view prints the outputs of the decoder process. Colors are added to highlight the lines that have special warning level: informations are in blue, warnings in orange and errors in red.



Toolbar

Icon Name Description
"Save as" button Save the content of this view in a text file. It opens a simple file dialog to choose the file to save the text to.
Clear button Clears the view.
"Scroll lock" button Locks the scrollbar, it deactivates the auto-scroll but let the data collection continue. Locking the scrollbar makes it easier to scroll in the view while collecting data.


Configuration

You can configure some display parameters for this view by opening the preferences window. Click on "Tools > Preferences..." to open the preferences panel. The preferences are applied at the beginning of a decoding session (e.g. will be applied the next time you will parse a trace).

  1. Max. lines in Console:

    Limit the output of the console view. Unlimited output can slow Trace Viewer if there are countless lines in the console output (more than 50000). When the maximum number of lines is reached the console is flushed keeping the last 5% lines that were in the console before the flush.



Tips

  1. Auto-scroll:

    Disable auto-scroll () to look at specific lines during the execution.

  2. Colored lines:

    Lines are colored differently depending on their meaning:

    • Red : error lines. Contain the word "error".
    • Orange: warning lines. Contain the word "warning" or start with "***".
    • Blue : info lines. Contain the word "info".
    • Green : TraceServer process output. Start with "TraceServer:".
    • Black : all other lines.
