Abstract

This document describes the Application Programming Interfaces (APIs) and software use within the application domain for a specific platform delivery. It also describes how the application domain gains access to the platform functionality. Generally updated for each customer release, the information in this document reflects the current state of the software under development in contrast to the functionality targeted for the final release. Written primarily for software engineers, designers and implementers, this document helps readers familiarize themselves with the software interfaces implemented within a specific customer delivery.
