How to create an authentication sign package for the U5500/U8500/M7400/L9540 platforms

An authentication sign package is a container that is used by the Sign Server to determine which key and certificate to use when signing an authentication challenge.

Pre-requisites

Please make sure that you have access to a U5500/U8500/M7400/L9540 authentication certificate

Instructions

Open the Authentication Sign Package Wizard by completing the following steps:

  • In the Eclipse menu, select File -> New > Other... (or press Ctrl+N)
  • Expand the Flash Kit category to make the Sign Packages category visible
  • Expand the Sign Packages category to see the Authentication Sign Package
  • Select the Authentication Sign Package and press Next

The following dialog will pop-up and the user must fill in correct values in order create the sign package

Required Fields:

  • Alias: The name of the sign package. This is used as input when signing software using the sign tool
  • Revision: The sign package revision
  • Description: The sign package description
  • Authentication Certificate: The certificate to include in the sign package. Note that the certificate must be signed by Sign Tool before it can be used in an authentication sign package
  • Output file: The path to the file where to write the sign package. The recommended file extension is .pkg
