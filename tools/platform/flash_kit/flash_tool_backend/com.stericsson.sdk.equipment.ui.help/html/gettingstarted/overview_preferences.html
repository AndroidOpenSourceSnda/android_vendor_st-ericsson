Flash Kit Preferences

Equipment Editor Preferences

Close editor when equipment is disconnected
Select whether to close the equipment editor or not when the associated mobile equipment is disconnected.

Flash Tool Backend Preferences

Configuration File
The path to an external mesp configuration file. Configuration file should have .mesp extension (e.g. backend.mesp).

Profiles Path
The path to where the flash tool backend profiles are stored. Profiles should have .prfl extension (e.g. profile-STE_DBX500_flashloader.prfl).

Loaders Path
The path to where the flash and customization loaders are stored. Loaders should have .ldr extension (e.g. loader.ldr).

LCD Library
The absolute path to the loader communication driver. Depending on operating system this can either be an .so file or a .dll file. This file is required to enable loader communication. To allow mobile equipment connections, select 'Accept Connections' in the Flash Tool Backend submenu of the Connected Equipment View drop-down menu.

LCM Library
The absolute path to the loader communication module. Depending on operating system this can either be an .so file or a .dll file. This file is required to enable loader communication. To allow mobile equipment connections, select 'Accept Connections' in the Flash Tool Backend submenu of the Connected Equipment View drop-down menu.

Remote Connection Port
This is the port to where remote clients (such as the Flash Tool CLI) will be allowed to connect. To allow remote connections, select 'Accept Remote Clients' in the Flash Tool Backend submenu of the Connected Equipment View drop-down menu.

Development mode
Preferences are switched to development mode if the grey-out settings were defined during startup (e.g. as parameters in flashkit.ini). If the flashkit.ini contains setup of configuration paths (e.g. path to backend.mesp, loaders, profiles), these may be altered only within the flashkit.ini file. If these paths are removed from the flashkit.ini file, the development mode is turned off and the use may define his own paths via Preference Page.

Signing Preferences

Use local signing
Check this checkbox to enable usage of local sign packages and keys. Note that this will disable usage of a sign server.

Use common folder
Check this checkbox to enable usage of common folder for sign packages and keys. Note that this will disable usage of a mirror structure for keys and packages.

Sign server
Specify the address to the sign server on the form <host>:<port>. Multiple sign servers can be separated by semicolons. Note that the sign servers will be tested in the order that they are written. The first successful connection will be used.

Sign package root (Linux only)
To enable signing in a Linux environment you must mount the sign server's share on your local machine. If the sign server packages are on a share called \\myShare\sharedFolder\signpackages\, then mount \\myShare\sharedFolder to a folder on your machine and specify this local folder as sign package root.

Local sign package root
The path to the locally stored sign packages that will be used if local signing is enabled.

Local key root
The path to the locally stored keys that will be used if local signing is enabled.

Local common folder for keys and packages root
The path to the common folder where keys and packages are locally stored that will be used if local signing is enabled.

Use default sign package
Check this checkbox to enable the usage of a default sign package.

Default sign package
When signing a file from the right-click menu, the sign package specified here will be used if the usage of a default sign package is enabled. This sign package will also be preselected in the signing wizard.

Dump Handling Preferences

Enable Automatic Download of Dump
Check this checkbox to enable automatic download of core dump when a equipment is dumping.

Download Path
The path to a directory where automatic download of dump will store the core dump file.

Enable Automatic Delete of Dump After Download
Check this checkbox to enable automatic delete of core dump file after automatic download of core dump happens.
